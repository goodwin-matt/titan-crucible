## Project Overview
This is a AI Python project managed with `uv`. It is designed to be highly modular, secure, and thoroughly tested.

## Testing Instructions
*   **Test Framework**: Always use `pytest`.
*   **Test Location**: Place all tests under the `tests/` directory.
*   **Test Naming**: Test files must be prefixed with `test_` (e.g., `tests/test_core.py`), and test functions must start with `test_`.

## Code Style Guidelines
*   **Python Standard**: Follow PEP 8 guidelines.
*   **Type Hinting**: Use type hints for all function signatures and public APIs to ensure clarity and safety.
*   **Formatting**: Use standard ruff for clean, uniform code layout.
*   **Documentation**: Provide clear docstrings (Google style) for all modules, classes, and public functions. Keep existing comments and docstrings intact unless updating them is a core part of the task.
